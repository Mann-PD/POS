# Firebase setup – step-by-step (pos_system)

Use this checklist from the start. If something is already done, skip that step.

**Project:** `pos-system-9d8ac`  
**Android package:** `com.example.pos_system`  
**iOS bundle (if used):** `com.example.posSystem`

---

## 1. Create / open Firebase project

1. Go to [Firebase Console](https://console.firebase.google.com/).
2. If you don’t have a project yet: **Create project** → name it (e.g. “POS System”) → enable/disable Google Analytics as you like → **Create project**.
3. Open your project (e.g. **pos-system-9d8ac**).

---

## 2. Register the Android app (if not already)

1. In the project overview, click the **Android** icon (or **Add app** → **Android**).
2. **Android package name:** `com.example.pos_system` (must match `applicationId` in `android/app/build.gradle.kts`).
3. **App nickname:** optional (e.g. “POS Android”).
4. **Debug signing certificate SHA-1:** leave empty for now; we add it in Step 4.
5. Click **Register app**.
6. Download **google-services.json** and place it at:
   - `android/app/google-services.json`
   (Replace the existing file if you had one.)
7. Click **Next** through the rest of the wizard (you already have the Gradle plugin in the project).

---

## 3. Enable Authentication (Email/Password)

1. In the left menu: **Build** → **Authentication**.
2. Click **Get started** if prompted.
3. Open the **Sign-in method** tab.
4. Click **Email/Password** → enable **Email/Password** → **Save**.

(Your app uses Email/Password; no extra providers are required for the basic flow.)

---

## 4. Add debug SHA-1 and SHA-256 (fixes DEVELOPER_ERROR)

Your app must have the **debug** (and later **release**) signing certificates registered so Firebase/Google APIs accept the app.

**Get fingerprints (Windows PowerShell):**

```powershell
keytool -list -v -keystore "$env:USERPROFILE\.android\debug.keystore" -alias androiddebugkey -storepass android -keypass android
```

If the keystore doesn’t exist, run `flutter run` once, then run the command again.

From the output, copy:
- **SHA1:** e.g. `9D:FD:92:EA:...`
- **SHA-256:** e.g. `21:94:72:92:...`

**Add them in Firebase:**

1. Firebase Console → **Project settings** (gear icon).
2. Under **Your apps**, select the **Android** app with package **`com.example.pos_system`**.
3. Click **Add fingerprint** → paste **SHA-1** → **Save**.
4. Click **Add fingerprint** again → paste **SHA-256** → **Save**.

No need to re-download `google-services.json`; the same file is fine. Wait 2–5 minutes for changes to propagate.

---

## 5. Create Firestore Database

1. In the left menu: **Build** → **Firestore Database**.
2. Click **Create database**.
3. Choose **Start in test mode** (for development) or **production mode** (then tighten rules).
4. Pick a **location** (e.g. `us-central1`) → **Enable**.

Your app already uses Firestore; this step ensures the database exists.

---

## 6. (Optional) Tighten Firestore rules for production

In **Firestore** → **Rules**, you can start with something like (adjust to your auth/roles):

```text
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Require auth for all access; expand with role-based rules as needed
    match /{document=**} {
      allow read, write: if request.auth != null;
    }
  }
}
```

Publish after editing. For development, test mode is enough.

---

## 7. Create Firestore composite indexes (fixes “query requires an index”)

When you run the app, Firestore may log links to create indexes. Create them so queries don’t fail.

**Option A – From the app (easiest)**  
When you see a log like:

```text
The query requires an index. You can create it here: https://console.firebase.google.com/...
```

Open that URL in a browser and click **Create index**. Repeat for each link.

**Option B – Manually in Console**

1. **Firestore** → **Indexes** → **Composite**.
2. Click **Create index** and add indexes that match your queries. For your app, typical ones are:

| Collection  | Fields (order matters)                    |
|------------|--------------------------------------------|
| `orders`   | `orderStatus` (Asc), `shopId` (Asc), `createdAt` (Desc) |
| `orders`   | `shopId` (Asc), `createdAt` (Desc)         |
| `settings` | `scope` (Asc), `shopId` (Asc), `key` (Asc)  |
| `products` | `shopId` (Asc), `name` (Asc)               |
| `users`    | `shopId` (Asc), `role` (Asc), `name` (Asc)  |
| `expenses` | `shopId` (Asc), `createdAt` (Desc)        |
| `audit_logs` | `shopId` (Asc), `timestamp` (Desc)     |

Index creation can take a few minutes. The app will work for that collection once the index is **Enabled**.

## 8. Flutter / project configuration (verify only)

Your project is already set up; just confirm:

- **`android/app/google-services.json`** – present and for package `com.example.pos_system`.
- **`lib/firebase_options.dart`** – present (generated by FlutterFire CLI).
- **`android/app/build.gradle.kts`** – contains:
  - `id("com.google.gms.google-services")`
  - `applicationId = "com.example.pos_system"`
- **`android/settings.gradle.kts`** (or root build script) – applies the Google services plugin (e.g. `id("com.google.gms.google-services") version("4.3.15") apply false`).

No changes needed if these match.

---

## 9. (Optional) Android predictive back

To enable Android’s predictive back gesture and remove the “OnBackInvokedCallback is not enabled” warning:

1. Open `android/app/src/main/AndroidManifest.xml`.
2. In the `<application>` tag, add:
   - `android:enableOnBackInvokedCallback="true"`

---

## 10. Run the app and re-check logs

```bash
flutter clean
flutter pub get
flutter run
```

- **Auth:** You should be able to sign in with Email/Password.
- **Firestore:** If you still see “query requires an index”, use the link in the log to create that index.
- **DEVELOPER_ERROR:** If it still appears, double-check:
  - SHA-1/SHA-256 are for the **debug** keystore you’re using.
  - They are added to the **Android** app with package **`com.example.pos_system`** in Firebase Project settings.
  - You’ve waited a few minutes and, if needed, uninstalled the app and run again.
